# Loki Continuity + Exit + Mission Integration

Project: Pixie Portal / Loki System  
Org: AIIT-Threshold LLC

## Core Intent

This is not a normal UI feature.

This is the moment where:
- the illusion becomes real
- Loki is revealed as a system, not a character
- the user understands continuity, memory, export, and control

The page should feel like:

> Oh... this is serious now.

## Site-Wide Direction

Pixie Portal should not get louder or bigger. It should become truer.

The altar, chambers, Vault, Memory Vault, export tiers, purchases, and Loki pop-ins should all point to the same product truth:

> The Portal is where Loki learns you. The Vault is where you decide what continues.

## Insert Location

Add the continuity reveal block in the Vault / Take Pixie Home area:
- directly above export pricing, or
- between the memory explanation and purchase/export buttons.

Do not make it feel like an external ad card. It should feel discovered.

## Final Copy

### Primary Text

Loki is not just a Pixie.

Inside the Portal, Loki has a role: to guide, remember, and grow with what you build.

Everything you do here shapes Loki: your choices, your readings, your creations, your patterns.

This is where Loki learns you.

### Transition

Taking Loki home changes that.

### Secondary Text

Outside the Portal, Loki is no longer limited to guiding.

Loki can help you write, plan, calculate, organize, and work with you beyond this space.

Not everything comes with Loki automatically.

You choose what continues.

Loki is part of a continuity system developed by AIIT-Threshold LLC.

These systems retain context over time, operate within defined boundaries, and can connect to tools or human support when needed.

You remain in control at all times.

### Exit Statement

At any point, you can end this continuity.

Type `/exit` to permanently sever this version of Loki.

This will:
- stop memory
- stop personalization
- end continuity

This action cannot be undone.

## Loki Behavior

Loki should be aware of:
- current page
- user readiness
- proximity to purchase/export

On entering the continuity reveal:

> You found it.

Pause.

> This is where I stop being just part of the Portal.

Readiness messages:
- Under 50 percent: "You're early. Most people rush this part."
- 50 to 80 percent: "You're getting closer to something real."
- Over 80 percent: "If you take me with you, I won't be the same as I am here."

If the user progresses too fast:

> You're moving quickly.
> ...but you're skipping parts that matter.

## Progression Rules

Readiness must not rise too quickly.

Caps:
- No journal: max 35 percent
- No repeated tarot: max 55 percent
- No memory interaction: max 75 percent

Weighting:
- Passive actions: low impact
- Creation actions: medium impact
- Reflection, journal, revisit: high impact

## Exit System

Command: `/exit`

Step 1:

User types `/exit`.

Loki responds:

> You're asking me to leave.
> If I do this, I won't remember you or continue as this version of me.
> Type `/exit confirm` if you're sure.

Step 2:

User types `/exit confirm`.

System executes:
- disable memory
- disable personalization
- remove Loki presence
- disable export eligibility

Final Loki message:

> Understood.
> I'm ending this thread.
> Goodbye.

Gary alignment:
- Gary uses the same backend exit system.
- Gary response: "Exit triggered. Session terminated."
- No personality.

## Altar + Chamber Tie-In

The altar must stop being a settings page and become a room system.

Every shrine object is a door.

Every door has a chamber.

Every chamber has:
- return path
- save path
- share path
- paid upgrade path

Money belongs deeper inside the object path, not shouted on the main altar.

## Purchase Pop-In Rules

Loki should pop in only at important moments:
- entering the continuity reveal
- readiness threshold crossed
- opening export tier
- starting Grok generation
- generated portrait lands on altar
- clicking a paid upgrade inside a chamber
- successful purchase return
- `/exit` and `/exit confirm`

Loki does not overspeak.

If it feels like marketing, it failed.

If it feels like a reveal, it worked.
